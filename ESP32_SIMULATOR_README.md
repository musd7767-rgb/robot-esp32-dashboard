# ESP32 Simulator - اختبار الموقع بدون ESP32 حقيقية

هذا السكريبت يحاكي سلوك ESP32 حقيقية بشكل واقعي جداً، مما يسمح لك باختبار الموقع والاتصال بـ Adafruit IO قبل شراء ESP32 حقيقية.

## المميزات

✅ **محاكاة واقعية**: يحاكي سلوك الروبوت الحقيقي مع دورات نشاط المحرك  
✅ **بيانات ديناميكية**: الجهد والتيار والحرارة تتغير بناءً على حمل المحرك  
✅ **دعم Adafruit IO**: يرسل البيانات مباشرة إلى Adafruit IO  
✅ **سهل الاستخدام**: بدون تكوين معقد  
✅ **توثيق كامل**: شرح مفصل لكل خطوة

## المتطلبات

### 1. تثبيت Python (إذا لم يكن مثبتاً)
```bash
# على Ubuntu/Debian
sudo apt-get install python3 python3-pip

# على macOS
brew install python3

# على Windows
# حمّل من https://www.python.org/downloads/
```

### 2. تثبيت المكتبات المطلوبة
```bash
pip install adafruit-io requests
```

## الخطوات الأولى

### الخطوة 1: إنشاء حساب على Adafruit IO

1. اذهب إلى [https://io.adafruit.com](https://io.adafruit.com)
2. أنشئ حساباً جديداً (أو سجل الدخول إذا كان لديك حساب)
3. اذهب إلى **My Key** في الأعلى اليمين
4. انسخ:
   - **Username**: سيكون شيء مثل `your_username`
   - **Active Key**: هذا هو `AIO_KEY`

### الخطوة 2: إنشاء Feeds على Adafruit IO

تحتاج إلى إنشاء feed لكل روبوت:

1. اذهب إلى **Feeds** في القائمة الجانبية
2. انقر على **Create a New Feed**
3. أنشئ feed باسم: `robot-master-data`
4. أنشئ feed آخر باسم: `robot-follower-data`

### الخطوة 3: تشغيل السكريبت

#### الخيار 1: بدون إرسال إلى Adafruit IO (عرض البيانات فقط)
```bash
python3 esp32_simulator.py
```

#### الخيار 2: إرسال إلى Adafruit IO
```bash
python3 esp32_simulator.py \
  --aio-username YOUR_USERNAME \
  --aio-key YOUR_AIO_KEY
```

#### الخيار 3: إرسال بفترات زمنية مختلفة
```bash
# إرسال كل ثانيتين
python3 esp32_simulator.py \
  --aio-username YOUR_USERNAME \
  --aio-key YOUR_AIO_KEY \
  --interval 2

# تشغيل لمدة دقيقة واحدة فقط
python3 esp32_simulator.py \
  --aio-username YOUR_USERNAME \
  --aio-key YOUR_AIO_KEY \
  --duration 60
```

## الخيارات المتاحة

```
--aio-username USERNAME     اسم مستخدم Adafruit IO
--aio-key KEY              مفتاح Adafruit IO
--interval SECONDS         الفترة الزمنية بين الإرسالات (افتراضي: 1 ثانية)
--duration SECONDS         مدة التشغيل بالثواني (افتراضي: بدون حد)
--http                     استخدام HTTP REST API بدلاً من مكتبة العميل
```

## مثال عملي كامل

```bash
# 1. تشغيل السكريبت لمدة 5 دقائق
python3 esp32_simulator.py \
  --aio-username my_username \
  --aio-key aio_1234567890abcdef \
  --interval 1 \
  --duration 300

# 2. افتح الموقع في المتصفح
# http://localhost:3000

# 3. اختر اللغة
# 4. اختر "المعلومات"
# 5. شاهد البيانات تتحدث في الوقت الفعلي!
```

## كيف يعمل السكريبت

### دورة النشاط (Activity Cycle)
السكريبت يحاكي دورة نشاط واقعية للروبوت:

1. **التسارع** (5 ثوان): المحرك يبدأ التسارع
2. **السرعة الكاملة** (5 ثوان): المحرك يعمل بسرعة كاملة
3. **التباطؤ** (5 ثوان): المحرك يبدأ بالتباطؤ
4. **التوقف** (5 ثوان): المحرك متوقف

### التأثيرات الفيزيائية المحاكاة

- **الجهد**: ينخفض مع زيادة حمل المحرك
- **التيار**: يزداد مع حمل المحرك
- **الحرارة**: تزداد مع حمل المحرك
- **البطارية**: تنخفض ببطء بمرور الوقت
- **الروبوت التابع**: يتبع نمط الروبوت القائد بنسبة أقل

## استكشاف الأخطاء

### المشكلة: "ModuleNotFoundError: No module named 'Adafruit_IO'"
**الحل**: ثبّت المكتبة:
```bash
pip install adafruit-io
```

### المشكلة: "Connection refused" أو "Failed to connect"
**الحل**: تأكد من:
1. اتصالك بالإنترنت
2. اسم المستخدم والمفتاح صحيحين
3. أنك أنشأت الـ feeds على Adafruit IO

### المشكلة: البيانات لا تظهر على الموقع
**الحل**:
1. تأكد من تشغيل السكريبت
2. تأكد من أن الموقع يتصل بـ Adafruit IO (سيتم شرح هذا في المرحلة التالية)
3. حاول تحديث الصفحة (F5)

## ملاحظات مهمة

📝 **الكود قابل للتخصيص** - يمكنك تعديل `esp32_simulator.py` لمحاكاة سيناريوهات مختلفة.
